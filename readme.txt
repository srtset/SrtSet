
        **********************************************************************************************
        SrtSet V1.2 一个整理字幕的小工具.
        
        数据库位于软件文件夹下的[data/srt.db],请保护好你的数据.
        
        有问题可以发邮件至braum@sina.com
        
        你尽管提bug,修一个算我输.
        
        以后升级软件,只需要替换软件文件夹内的[SrtSet.exe]即可.
        

        **********************************************************************************************
        大致工作流程:
        
        1.遍历文件夹下面的文件,判断文件扩展名是否是[ass srt ssa vtt].如果不是,移入[SrtSet未处理文件]文件夹.
        
        2.尝试解析文件名,如果不能依据文件名获取番号,就把文件移入[SrtSet未识别文件]文件夹.
        
        3.尝试将[ass ssa vtt]文件转成srt格式,尝试解析最终的srt内容.如果失败,移入[SrtSet未解析文件]文件夹.
        
        4.将成功解析的srt,通过整理,繁体转简体,写入数据库.将成功解读的文件移入[SrtSet已处理文件]文件夹.
        
        5.如果数据库中有相同番号的字幕,最终会保留字幕行数最多的一个.
        

        **********************************************************************************************
        功能详细说明:
        
        0:阅读说明.关闭说明文档可返回SrtSet
        
        1:整理字幕.选择文件夹,整理文件夹下的字幕
        
        2:导出字幕.输入番号,依据番号导出数据库中相关字幕
        
        3:所有字幕.导出数据库中的所有字幕
        
        4:附加整理.选择文件夹,整理该文件夹下其他三个未成功处理的文件夹
            通过遍历其他三个未成功处理的文件夹,解析文件名获取番号
            如果数据库中存在该番号,就移入[SrtSet已处理文件]文件夹
            如果不存在,文件不移动
        
        5:手动整理.借助软件,手动规整字幕文件名称.关闭小工具可返回SrtSet
        
        6:还原文件.将整理的文件夹还原
            通过整理文件夹下的字幕,最终会转移动该文件夹下的四个子文件夹中
            如果想还原成整理前的状态,使用此功能
        
        7:修改规则.修改完成后可以选择一个文件进行测试
            如果是直接修改[config/regex.ini]文件，需要重启程序才能生效
            如果修改规则后造成程序崩溃,只需要删除配置文件,程序重启后重置规则
            =====================================================
            对番号的解析,默认番号由[前缀] + [分割线] + [后缀]组成
            比如'ABP' + '-' + '001'; '1Pondo' + '_' + '092020_001'
            程序内置解析规则:由'字母'组成的[前缀],'-'或者'00'或者'_'或者''组成的分割线,'数字'组成的后缀
            用户也可以配置程序目录下的[config/regex.ini]的文件.这需要你懂一点正则表达式.
            当然这一切的前提是建立在你的字幕文件名中的番号比较规范.
            特殊规则:
                [1Pondo]            # 关键字
                pre = 1Pondo        # 前缀
                sep = _             # 分割线
                suf = \d{6}_\d{3}   # 后缀
            程序先判断文件名中是否有特殊规则的关键字:
            如果有,就按照特殊规则中后缀的正则提取的后缀
                最后按照 特性规则的前缀+特殊规则的分割线+提取的后缀 组成文件名.
            如果没有,就按照内置解析规则去提取前缀,然后再判断是否在 SrtSetNormalRuleSection 中.
                然后再按照 SrtSetNormalRuleSection 中对应的数字位数,进行后缀的提取.最后组成文件夹.
            
        
        8:重置规则.重置正则配置文件
        
        9:退出程序
